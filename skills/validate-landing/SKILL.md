# /validate-landing

가설 문서를 기반으로, 마케팅 심리학 원칙이 적용된 스모크테스트 랜딩페이지 HTML을 생성한다.

## 트리거

사용자가 다음과 같이 말할 때:
- "랜딩 만들어줘", "랜딩페이지 생성", "검증 페이지 만들어줘"
- "validate landing", "smoke test page"
- `/validate-landing`

## 사전 조건

`docs/hypotheses/[name].md` 파일이 존재해야 한다.
없으면 → "먼저 `/validate-hypothesis`로 가설을 정리해주세요." 안내 후 종료.

가설 문서가 여러 개면 → 목록을 보여주고 선택하게 한다.

## 워크플로우

### Step 1: 가설 문서 로드

선택된 가설 문서에서 다음을 추출한다:
- 한 줄 요약
- Problem (핵심 문제)
- Customer Segment (타겟)
- Unique Value Proposition (UVP)
- Existing Alternatives (기존 대안)
- Price Point (가격대)
- Riskiest Assumption (핵심 가설)

### Step 2: 랜딩 전략 수립

사용자에게 추가 질문한다:

```
Q1. 랜딩의 CTA는 무엇인가요?
    (a) 이메일/웨이트리스트 등록
    (b) 가격표 확인 → 사전결제
    (c) 무료 체험 신청
    (d) 상담 신청
    (e) 기타

Q2. 브랜드명 또는 제품명이 있나요? (없으면 임시로 만들어드립니다)

Q3. 톤앤매너는? (기본: 전문적이면서 친근한)
```

### Step 3: 심리학 기반 카피라이팅

아래 6가지 원칙을 **모든 랜딩페이지에 자동 적용**한다.
사용자가 마케팅 이론을 몰라도 스킬이 알아서 적용한다.

---

#### 원칙 1: 앵커링 (Anchoring Effect)

**적용 위치**: 히어로 섹션
**방법**: 기존 대안의 비용/시간/고통을 먼저 제시하고, 우리 솔루션을 대비시킨다.

```
나쁜 예: "월 9,900원에 모든 기능을 사용하세요"
좋은 예: "매달 30만원씩 쓰던 [기존 방식]을 월 9,900원으로 대체하세요"
```

가설 문서의 `Existing Alternatives`와 `Price Point`를 사용하여 앵커를 설계한다.

#### 원칙 2: AIDA 흐름 (Attention → Interest → Desire → Action)

**적용 위치**: 페이지 전체 스크롤 구조

```
섹션 1 (Attention)  — 히어로: 문제를 한 문장으로 찔러서 멈추게 함
섹션 2 (Interest)   — 가치 제안: "이걸 어떻게 해결하는지" 3개 불릿
섹션 3 (Desire)     — 사회적 증거 + 구체적 결과: 갖고 싶게 만듦
섹션 4 (Action)     — CTA: 지금 행동하게 만듦
```

#### 원칙 3: 희소성 + FOMO (Fear Of Missing Out)

**적용 위치**: CTA 섹션 주변
**방법**: 수량 제한, 시간 제한, 얼리버드 가격 중 최소 1개를 포함한다.

```
예시:
- "얼리버드 50명 한정"
- "사전등록 기간: ~[날짜]"
- "런칭 후 가격이 변경됩니다"
```

#### 원칙 4: 전제 효과 (Presupposition)

**적용 위치**: CTA 카피
**방법**: 미래를 기정사실화하는 문장을 사용한다. "~할 수도 있다"가 아니라 "~되기 전에"로 쓴다.

```
나쁜 예: "가격이 오를 수도 있으니 지금 등록하세요"
좋은 예: "가격이 변경되기 전에 지금 확인하세요"
```

#### 원칙 5: 손실 회피 (Loss Aversion)

**적용 위치**: CTA 주변 서브카피
**방법**: 얻는 것보다 잃는 것을 강조한다. 손실의 고통은 이득의 기쁨보다 2배 강하다.

```
나쁜 예: "등록하면 할인 혜택을 받습니다"
좋은 예: "지금 등록하지 않으면 얼리버드 혜택을 놓치게 됩니다"
```

#### 원칙 6: 넛지 (Nudge)

**적용 위치**: CTA 버튼 텍스트
**방법**: 강압적 명령("지금 구매하세요") 대신 부드러운 권유를 사용한다.
선택 주도권을 사용자에게 넘기는 느낌을 준다.

```
나쁜 예: "지금 바로 결제하세요"
좋은 예: "무료로 먼저 살펴보기"
         "얼리버드 혜택 확인하기"
         "어떤 플랜이 맞는지 보기"
```

---

### Step 4: HTML 생성

**단일 HTML 파일**로 생성한다. 외부 의존성 없이 인라인 CSS만 사용.

**파일 경로**: `docs/hypotheses/[slug]-landing.html`

**필수 구성요소**:

```
[1] 히어로 섹션
    - 앵커링 적용된 헤드라인
    - 문제를 찌르는 서브헤드
    - 히어로 CTA 버튼

[2] 가치 제안 섹션
    - 3개 불릿 (아이콘 + 제목 + 설명)
    - 가설 문서의 UVP 기반

[3] 사회적 증거 / 상세 설명 섹션
    - 구체적 수치나 비교
    - 희소성 신호

[4] 가격/플랜 섹션 (선택: CTA가 결제형일 때)
    - 가격표
    - 앵커링: 기존 대안 가격 취소선 표시

[5] 최종 CTA 섹션
    - 전제 효과 + 손실 회피 카피
    - 넛지형 버튼 텍스트
    - 이메일 입력 폼 또는 결제 링크

[6] Thank You 페이지 (같은 HTML 내 hidden 섹션)
    - CTA 클릭 시 메인 숨기고 Thank You 표시
    - 전환 추적용 (광고 플랫폼에서 이 URL 앵커를 전환으로 설정)
```

**디자인 기본값**:
- 모바일 퍼스트 (max-width: 640px 기준)
- 폰트: system-ui
- 컬러: 흰 배경, 어두운 텍스트, CTA는 눈에 띄는 강조색
- 깔끔하고 신뢰감 있는 톤

**Thank You 페이지 구현**:
```html
<!-- 메인 콘텐츠 -->
<div id="main-content">...</div>

<!-- Thank You (기본 숨김) -->
<div id="thank-you" style="display:none;">
  <h2>감사합니다!</h2>
  <p>등록이 완료되었습니다.</p>
</div>

<script>
document.getElementById('cta-form').addEventListener('submit', function(e) {
  e.preventDefault();
  document.getElementById('main-content').style.display = 'none';
  document.getElementById('thank-you').style.display = 'block';
  window.location.hash = 'thank-you';
});
</script>
```

이 `#thank-you` 해시를 광고 플랫폼에서 전환 URL로 설정하면 별도 애널리틱스 없이 전환 추적이 가능하다.

### Step 5: 사용자 확인 및 수정

생성된 HTML을 브라우저에서 열어볼 수 있도록 경로를 안내한다.
수정 요청이 있으면 반영한다.

**안내 메시지**:
```
✅ 랜딩페이지가 생성되었습니다.

📄 파일: docs/hypotheses/[slug]-landing.html
🔍 브라우저에서 열어서 확인해보세요.

📊 전환 추적 설정:
   광고 플랫폼에서 전환 URL을 [배포URL]#thank-you 로 설정하면
   별도 애널리틱스 없이 전환 추적이 됩니다.

🚀 다음 단계:
   1. 이 HTML을 원하는 곳에 배포 (Vercel, Netlify, Carrd 등)
   2. 광고 집행 ($200~500, 최소 300명 트래픽)
   3. 데이터 수집 후 → /validate-judge 실행
```

## 주의사항

- 외부 CDN이나 라이브러리 의존 금지. 단일 HTML + 인라인 CSS/JS만 사용.
- 심리학 원칙 6개는 **항상 전부 적용**한다. 선택이 아니라 기본값이다.
- 가짜 사회적 증거(허위 후기, 가짜 숫자)를 만들지 않는다. 없으면 생략.
- 디자인은 과하지 않게. 검증용이므로 "충분히 좋은" 수준이면 된다.
- 폼 제출 시 실제 데이터 전송은 포함하지 않는다. 사용자가 배포 환경에 맞게 연결한다.
